AL-/BC-Regeln zur Qualitätssicherung im Entwicklungsbereich

AL und BC-Programmierregeln sind grundlegende Richtlinien, die uns helfen Kunden sauberen, wartbaren und vor allem effizienten Code zu bieten. Durch sie werden Best Practices wie Lesbarkeit und Modularität gefördert.

Regeln

Hier findet ihr die wichtigsten Regeln!

Projektstruktur

  • Quellcode gehört in den SRC-Ordner
  • Für Translation Files wird ein dedizierter Ordner angelegt

Code

  • Tests für alle möglichen Ergebnisse
  • Ein Leerzeichen nach jedem Namespace
  • Globale Varibalen werden mit einem _ gekennzeichnet
  • Keine Leerzeichen in Varibalen, Objektnamen und Methoden
  • VAR | Using wird alphabetisch sortiert
  • Regeln aus den AL-Guidelines welche hier gefunden werden können

DevOps und PRs

  • Es müssen Code Reviews gemacht werden
  • TestFiles als Anhang beifügen oder einen Link welcher zu den TestFiles führt

Updates

Jeder kann sich freiwillig bei dieser Webpage beteiligen. Dazu muss derjenige nur auf das GitHub Repo navigieren, das Repo pullen, anschließend so wie in der ReadMe einen Eintrag einrichten und einen Pull Request erstellen. Danach wird der PR sobald wie möglich bearbeitet und je nach Qualität gehandhabt.
